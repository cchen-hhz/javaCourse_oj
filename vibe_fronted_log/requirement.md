# 前端项目要求

你在制作一个前后端分离的 online judge 项目，后端已经写好了，你需要完成前端部分。

请在阅读后先按要求生成一个 `plan.md` 文档，在我审查后给出进一步指令。

## 技术栈选择

你需要使用 `astro` 和 `vue3` 为主要框架实现你的前端部分。

如果有附加的技术栈，请写在 `plan.md` 文档中。

## 结构要求

你需要有职责分明的结构设计，不要在一个文件中完成所有任务。你需要编写一个可维护性高，易于阅读，扩展的前端程序。

## API

所有的 API 可以在 backend/web/../controller 内找到，你可以浏览 backend 目录进一步了解。

如果你需要额外的 API，请写在 `plan.md` 中，我会视情况添加。

## 审美

请保持良好的审美。

## 登录管理

你还需要阅读 backend/security 部分，你需要修改 securityconfig 使得登录验证时自动跳转到 login/register 页面中

## 当前 API 的问题

当前 API 存在返回状态码不正确的问题，一些失败的请求会返回 500 错误，需要你来判别。
